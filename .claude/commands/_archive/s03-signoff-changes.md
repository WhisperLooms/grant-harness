# S03: Implement Signoff Changes

Track and implement changes from signoff review (Phase 2).

## Usage
`/s03-signoff-changes <company-id> <grant-id>`

## Tasks
1. Collect all reviewer comments
2. Categorize: critical, recommended, optional
3. Implement changes in application
4. Log change history (audit trail)
5. Re-request approval from relevant reviewers

## Change Tracking
- Who requested change
- What changed (before/after)
- When implemented
- Who approved change

## Output
Change log for compliance and audit

## Next
`/s04-signoff-finalize` when all approvals complete
